## 💻 Шаг 1: Подключение к хосту

### 🔑 Данные для авторизации
При подключении программа запросит у вас следующие данные (их можно найти в панели вашего хостинг-провайдера):
* **IP-адрес:** `IPv4` вашего сервера.
* **Логин:** обычно `root` (для ОС Ubuntu).
* **Пароль:** создается вами при покупке/настройке виртуального сервера.

## 🐳 Шаг 2: Установка Docker

После успешного подключения к консоли последовательно выполните следующие команды для настройки Docker.

### 1. Обновление индексов пакетов
```bash
sudo apt update
```

### 2. Установка зависимостей
```bash
sudo apt install apt-transport-https ca-certificates curl software-properties-common -y
```

### 3. Добавление GPG-ключа Docker
```bash
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
```

### 4. Подключение официального репозитория
```bash
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu focal stable"
```

### 5. Установка Docker CE
```bash
sudo apt install docker-ce -y
```

### 6. Проверка статуса службы
```bash
sudo systemctl status docker
```

> ✅ **Проверка:** Если служба работает корректно, в консоли вы увидите статус `active (running)`.

## 🎮 Шаг 3: Загрузка сервера

Когда Docker полностью готов к работе, скачайте официальный образ приватного сервера командой:

```bash
docker pull n0uk/dynast.io-server-private:release
```
Ссылка на оффициальный docker репозиторий разработчика игры: https://hub.docker.com/r/n0uk/dynast.io-server-private

---

### Создание папки сервера
Создайте папку с файлами сервера (к примеру папку `dyn_private` в директории `/root/`), скачайте файлы своего сервера (в качестве примера в репозитории размещенны стандартные файлы сервера), загрузите свою базу данных и файл `settings.env` ([Скачать settings.env](settings.env)
).

> ⚠️ **ВАЖНОЕ УТОЧНЕНИЕ**
> Чтобы избежать проблем с загрузкой файлов контейнером нужно загрузить базу данных сразу в папку сервера, хотя бы пустую.

### Настройка settings.env

| `DYNASTIO_PING_PORT` | порт нашего сервера / Установить пинг-порт |
| `DYNASTIO_SSL_PORT` | порт нашего сервера / Установить SSL-порт |

| `DYNASTIO_CONNECTIONS_LIMIT` | Установить лимит подключений |
| `DYNASTIO_REGION` | Установить регион |
| `DYNASTIO_LABEL` | Установить метку (название сервера) |
| `DYNASTIO_DEBUG` | Включить режим отладки (Debug) |
| `DYNASTIO_ADMIN` | ID администратора сервера |
| `DYNASTIO_CUSTOM` | Включить режим кастомного сервера |

## Шаг 4: HTTPS подлючение
Тут у нас 2 варианта развития: создание туннеля либо покупка домена, мы разберём оба варианта:

### Вариант 1: Туннелирование
Для туннелирования можно использовать разные сервисы. Лучшим является cloudflare, но т.к. он заблокирован в России используем российский анналог Cloudpub.

1. Заходим на сайт https://cloudpub.ru и проходим лёгкую регистрацию.
2. Устанавливаем cloudpub на наш VPS сервер:
   
   **2.1.** Откройте терминал Ubuntu и скачайте архив с актуальной версией утилиты для Linux:
   ```bash
   wget https://cloudpub.rudownload/stable/clo-3.1.0-stable-linux-x86_64.tar.gz
   ```
   **2.2.** Распакуйте скачанный архив:
   ```bash
   tar -xvf clo-3.1.0-stable-linux-x86_64.tar.gz
   ```
   **2.3.** Сделайте файл исполняемым и переместите его в системную директорию, чтобы команду clo можно было вызывать из любой папки:
   ```bash
   chmod +x clo
   sudo mv clo /usr/local/bin/
   ```

3. Авторизация в системе:
   ```bash
   clo login
   ```

4. Создаём туннель и ставим на автозапуск:
   
   **4.1.** Зарегистрируйте постоянный туннель:
   ```bash
   clo register http 80
   ```
   **4.2.** Создайте файл службы systemd (его можно скачать с репозитория), если логин вашего VPS не root, то поменяйте строку `User=root`, поменяйте root на имя пользователя вашего VPS.
   
   **4.3.** Активируйте и запустите службу:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable cloudpub.service
   sudo systemctl start cloudpub.service
   ```

> ℹ️ Имя вашего туннеля можно узнать при его создании, либо в личном кабинете в разделе публикации.

### Вариант 2: Покупка и настройка домена
Провайдеров, которые могут предоставить вам домен полно, в этом гайде используется РЕГ.РУ (https://www.reg.ru/)

1. **Покупка домена.** Купите любой доступный домен, самыми дешёвыми вариантами будут .ru и .online
2. **Настройка домена.** Перейдите в личный кабинет в раздел домены. Нажмите на свой домен, найдите DNS-серверы и управление зоной и нажмите изменить
3. **Добавление записи.** Перейдите ниже. Найдите кнопку добавить запись, из предложенных вариантов выберите AAAA. В поле subdomian введите желаемый поддомен. к примеру server. В поле Ip адреса введите айпи вашего VPS.
4. **Получение сертификата.** Выполните следующие команды: 
   
   **4.1.** Установка и обновление core:
   ```bash
   sudo snap install core; sudo snap refresh core
   ```
   **4.2.** Удаление старого certbot:
   ```bash
   sudo apt remove certbot
   ```
   **4.3.** Установка certbot через snap:
   ```bash
   sudo snap install --classic certbot
   ```
   **4.4.** Создание символической ссылки:
   ```bash
   sudo ln -s /snap/bin/certbot /usr/bin/certbot
   ```
   **4.5.** Получение сертификата:
   ```bash
   sudo certbot certonly --standalone -d your_domain
   ```
   *(где your_domian это имя вашего домена, при получении сертификата введите почту)*

5. **Установка nginx и настройка конфига** 
   
   **5.1.** Пропишите команду:
   ```bash
   sudo apt install nginx
   ```
   **5.2.** Перейдите в директорию `/etc/nginx/` и замените там конфиг ([Скачать конфигурацию Nginx](./nginx.conf)). В самом конфиге замените `your-domian` на ваш домен (к примеру `server.noahfamily.online`). Если вы запускаете сервер не на порте 80, но поменяйте в строке `server 127.0.0.1:80;` `80` на порт вашего сервера. Сохраните конфиг.
   
   **5.3.** Перезагрузите nginx командой:
   ```bash
   systemctl restart nginx
   ```
   и проверьте его статус командой:
   ```bash
   systemctl status nginx
   ```

Готово! Ваш домен настроен.

## Шаг 5: Запуск сервера
После того как вы настроили все свои файлы, домен и т.п. откройте `settings.env`

* Поменяйте `DYNASTIO_PORT`, `DYNASTIO_PING_PORT` если вы запускаете сервер на другом порте
* Поменяйте `DYNASTIO_SSL_HOST` и `DYNASTIO_IP` на ваш домен/ туннель
* Поменяйте `DYNASTIO_UID` и `DYNASTIO_PIN_CODE` ( на аккаунтах с премиумом лимит онлайна 50, а не 25)

Все остальные настройки были расписаны ранее.

### Команда запуска сервера 
```bash
docker run --rm --env-file /root/dyn_private/settings.env -e DYNASTIO_LABEL="Noahs-test" -p 800:800 -v /root/dyn_private/data.sqlite:/usr/local/bin/data.sqlite -v /root/dyn_private/settings:/usr/local/bin/settings -v /root/dyn_private/maps:/usr/local/bin/maps -v /root/dyn_private/logs:/var/log --name dyn -d n0uk/dynast.io-server-private:release
```

#### Описание команды: 
* `--env-file /root/dyn_private/settings.env` — путь к вашему .env файлу
* `-e DYNASTIO_LABEL="Noahs-test"` — название вашего сервера
* `-p 80:80` — открытие портов сервера (поменяйте если используете другие)
* `-v /root/dyn_private/data.sqlite:/usr/local/bin/` — путь к дб, картам, настройкам и логам, поменяйте первую часть настройки на путь к вашим файлам вплоть до `/root/dyn_private/`
* `--name dyn` — название docker контейнера

### Управление контейнером

* **Команда остановки контейнера:** 
  ```bash
  docker stop dyn
  ```
* **Команда перезагрузки контейнера:**
  ```bash
  docker restart dyn
  ```
  *(Где `dyn` - название вашего контейнера)*

* **Проверка запущенных контейнеров:**
  ```bash
  docker ps
  ```
