## 💻 Шаг 1: Подключение к хосту (для VPS) или Подготовка Windows (Локально)

Выберите ваш вариант размещения игрового сервера:

### Вариант А: Если вы используете удаленный сервер (Ubuntu VPS)
Для управления сервером вам понадобится SSH-клиент. Вы можете использовать любой удобный инструмент:

* **MobaXterm** — отличное решение «всё в одном» (содержит встроенную консоль и файловый менеджер SFTP для загрузки баз данных и конфигураций).
  
  <img width="1280" height="683" alt="Xterm" src="https://github.com" />

* **VS Code (Remote - SSH)** — официальный плагин для популярного редактора кода. В данном руководстве мы будем использовать именно его.
  
  <img width="1280" height="683" alt="VScode" src="https://github.com" />

#### 🔑 Данные для авторизации в VPS:
При подключении программа запросит у вас следующие данные (их можно найти в панели вашего хостинг-провайдера):
* **IP-адрес:** `IPv4` вашего сервера.
* **Логин:** обычно `root` (для ОС Ubuntu).
* **Пароль:** создается вами при покупке/настройке виртуального сервера.

---

### Вариант Б: Если вы используете домашний компьютер (Windows Локально)
Если вы запускаете сервер у себя на ПК, подключаться по SSH никуда не нужно. Вам потребуется настроить Windows под запуск контейнеров.

#### 🧠 Теория: Как устроен Docker Desktop и зачем ему WSL 2?
Контейнеры Docker изначально создавались под операционную систему Linux, поэтому они используют общие системные вызовы его ядра. Напрямую в среде Windows запустить Linux-контейнер технически невозможно.

**Docker Desktop** изящно решает эту проблему с помощью технологии **WSL 2 (Windows Subsystem for Linux)**:
* При установке Docker Desktop в вашей системе разворачивается скрытая, ультра-легкая и глубоко интегрированная виртуальная машина Linux (дистрибутив WSL).
* Когда вы вводите команду `docker run` в PowerShell на Windows, Docker Desktop незаметно перенаправляет этот запрос внутрь Linux-окружения WSL 2.
* Сервер игры выполняется внутри полноценного, изолированного Linux-контейнера, но вы можете управлять им через красивое графическое окно на Windows, а также привязывать папки напрямую со своего жесткого диска (например, `C:\dyn_private`).

## 🐳 Шаг 2: Установка Docker / Docker Desktop

Инструкции по подготовке среды для обеих операционных систем:

### 🐧 Инструкция по установке Docker на Ubuntu VPS

После успешного подключения к консоли удаленного хоста последовательно выполните следующие команды для настройки Docker.

#### 1. Обновление индексов пакетов
```bash
sudo apt update
```

#### 2. Установка зависимостей
```bash
sudo apt install apt-transport-https ca-certificates curl software-properties-common -y
```

#### 3. Добавление GPG-ключа Docker
```bash
curl -fsSL https://docker.com | sudo apt-key add -
```

#### 4. Подключение официального репозитория
```bash
sudo add-apt-repository "deb [arch=amd64] https://docker.com focal stable"
```

#### 5. Установка Docker CE
```bash
sudo apt install docker-ce -y
```

#### 6. Проверка статуса службы
```bash
sudo systemctl status docker
```

> ✅ **Проверка:** Если служба работает корректно, в консоли вы увидите статус `active (running)`.
> 
> <img width="1007" height="215" alt="docker stat" src="https://github.com" />

### 🪟 Инструкция по установке Docker Desktop на Windows

#### 1. Скачивание и установка программы
1. Скачайте официальный инсталлятор: [Docker Desktop для Windows](https://docker.com).
2. Запустите установщик. При установке обязательно убедитесь, что включена галочка **«Use WSL 2 instead of Hyper-V (recommended)»**.
3. Перезагрузите компьютер по требованию установщика.
4. Запустите Docker Desktop с рабочего стола и дождитесь, пока индикатор в левом нижнем углу окна программы станет зеленым (`Engine Running`).

#### 2. Подготовка и обновление WSL 2
Чтобы Docker Desktop работал стабильно и без сбоев, необходимо обновить ядро подсистемы Linux (WSL) в Windows до самой актуальной версии. Для этого откройте **PowerShell от имени администратора** и выполните команду:

```powershell
wsl.exe --update
```

После завершения обновления рекомендуется перезапустить Docker Desktop. Теперь ваша Windows полностью готова к запуску игровых Linux-контейнеров!

Когда Docker полностью готов к работе, скачайте официальный образ приватного сервера командой:

```bash

```
Ссылка на оффициальный docker репозиторий разработчика игры: https://hub.docker.com/r/n0uk/dynast.io-server-private

Создайте папку с файлами сервера (к примеру папку `dyn_private` в директории `/root/`), скачайте файлы своего сервера (в качестве примера в репозитории размещенны стандартные файлы сервера), загрузите свою базу данных и файл `settings.env` ([Скачать settings.env](settings.env)

## Шаг 4: HTTPS подлючение
Тут у нас 2 варианта развития: создание туннеля либо покупка домена, мы разберём оба варианта:

### Вариант 1: Туннелирование
Для туннелирования можно использовать разные сервисы. Лучшим является cloudflare, но т.к. он заблокирован в России используем российский анналог Cloudpub.

1. Заходим на сайт https://cloudpub.ru и проходим лёгкую регистрацию.

---

#### 🐧 Инструкция по настройке Cloudpub на Ubuntu VPS

2. Устанавливаем cloudpub на наш VPS сервер:
   
   **2.1.** Откройте терминал Ubuntu и скачайте архив с актуальной версией утилиты для Linux:
   ```bash
   wget https://cloudpub.ru
   ```
   **2.2.** Распакуйте скачанный архив:
   ```bash
   tar -xvf clo-3.1.0-stable-linux-x86_64.tar.gz
   ```
   **2.3.** Сделайте файл исполняемым и переместите его в системную директорию, чтобы команду clo можно было вызывать из любой папки:
   ```bash
   chmod +x clo
   sudo mv clo /usr/local/bin/
   ```

3. Авторизация в системе:
   ```bash
   clo login
   ```

4. Создаём туннель и ставим на автозапуск:
   
   **4.1.** Зарегистрируйте постоянный туннель:
   ```bash
   clo register http 80
   ```
   **4.2.** Создайте файл службы systemd (его можно скачать с репозитория), если логин вашего VPS не root, то поменяйте строку `User=root`, поменяйте root на имя пользователя вашего VPS.
   
   **4.3.** Активируйте и запустите службу:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable cloudpub.service
   sudo systemctl start cloudpub.service
   ```

> ℹ️ Имя вашего туннеля можно узнать при его создании, либо в личном кабинете в разделе публикации.

---

#### 🪟 Инструкция по настройке Cloudpub на Windows

Вы можете использовать любой удобный вариант запуска туннеля на локальной машине: через программу с интерфейсом или через командную строку.

##### Способ А: Через графическое приложение (GUI)
1. Скачайте официальное приложение с графическим интерфейсом для Windows со страницы личного кабинета Cloudpub.
2. Запустите скачанный файл `cloudpub.exe`.
3. Авторизуйтесь в программе, введя логин и пароль от вашего аккаунта.
4. Нажмите на кнопку **«Новая публикация»** (или «Добавить»).
5. Выберите тип ресурса: **HTTP**.
6. Укажите порт локального сервера: **80** (или измените на свой порт, если настраивали другой).
7. Нажмите на кнопку **«Опубликовать»**.
8. Программа отобразит ваш готовый URL-адрес туннеля. Скопируйте его. Не закрывайте приложение во время игры!

##### Способ Б: Через Командную строку (Консоль CLI)
1. Скачайте консольную версию утилиты для Windows и распакуйте архив в удобную папку (например, `C:\cloudpub\`).
2. Откройте **Командную строку** Windows (`Win + R` -> введите `cmd` -> Нажмите `Enter`).
3. Перейдите в папку с утилитой с помощью команды:
   ```cmd
   cd C:\cloudpub
   ```
4. Авторизуйтесь в системе:
   ```cmd
   clo login
   ```
5. Зарегистрируйте постоянный туннель, привязав его к порту локального сервера:
   ```cmd
   clo register http 80
   ```
6. Консоль выведет адрес вашего туннеля. Оставьте это окно открытым на все время работы сервера.

> ℹ️ Имя вашего туннеля можно узнать прямо в интерфейсе приложений при их создании, либо в личном кабинете на сайте в разделе публикации.

### Вариант 2: Покупка и настройка домена (для VPS)
Провайдеров, которые могут предоставить вам домен полно, в этом гайде используется РЕГ.РУ (https://www.reg.ru/)

1. **Покупка домена.** Купите любой доступный домен, самыми дешёвыми вариантами будут .ru и .online
2. **Настройка домена.** Перейдите в личный кабинет в раздел домены. Нажмите на свой домен, найдите DNS-серверы и управление зоной и нажмите изменить
3. **Добавление записи.** Перейдите ниже. Найдите кнопку добавить запись, из предложенных вариантов выберите A. В поле subdomian введите желаемый поддомен. к примеру server. В поле Ip адреса введите айпи вашего VPS
4. **Получение сертификата.** Выполните следующие команды: 
   
   **4.1.** Установка и обновление core:
   ```bash
   sudo snap install core; sudo snap refresh core
   ```
   **4.2.** Удаление старого certbot:
   ```bash
   sudo apt remove certbot
   ```
   **4.3.** Установка certbot через snap:
   ```bash
   sudo snap install --classic certbot
   ```
   **4.4.** Создание символической ссылки:
   ```bash
   sudo ln -s /snap/bin/certbot /usr/bin/certbot
   ```
   **4.5.** Получение сертификата:
   ```bash
   sudo certbot certonly --standalone -d your_domain
   ```
   *(где your_domian это имя вашего домена, при получении сертификата введите почту)*

5. **Установка nginx и настройка конфига** 
   
   **5.1.** Пропишите команду:
   ```bash
   sudo apt install nginx
   ```
   **5.2.** Перейдите в директорию `/etc/nginx/` и замените там конфиг ([Скачать конфигурацию Nginx](./nginx.conf)). В самом конфиге замените `your-domian` на ваш домен (к примеру `server.noahfamily.online`). Если вы запускаете сервер не на порте 80, то поменяйте в строке `server 127.0.0.1:80;` `80` на порт вашего сервера. Сохраните конфиг.
   
   **5.3.** Перезагрузите nginx командой:
   ```bash
   systemctl restart nginx
   ```
   и проверьте его статус командой:
   ```bash
   systemctl status nginx
   ```

Готово! Ваш домен настроен.

## Шаг 5: Запуск сервера
После того как вы настроили все свои файлы, домен и т.п. откройте `settings.env`

* Поменяйте `DYNASTIO_PORT`, `DYNASTIO_PING_PORT` если вы запускаете сервер на другом порте
* Поменяйте `DYNASTIO_SSL_HOST` и `DYNASTIO_IP` на ваш домен/ туннель
* Поменяйте `DYNASTIO_UID` и `DYNASTIO_PIN_CODE` ( на аккаунтах с премиумом лимит онлайна 50, а не 25)

> ⚙️ **Также обратите внимание:** При изменении портов в `settings.env`, их необходимо обязательно поменять и в файле конфигурации `server.json`:
> ```json
> "port": 80,
> "ping_port": 80,
> "ssl_port": 443,
> "ssl_ping_port": 443,
> ```

Все остальные настройки были расписаны ранее.

### Команда запуска сервера 
```bash
docker run --rm --env-file /root/dyn_private/settings.env -e DYNASTIO_LABEL="Noahs-test" -p 80:80 -v /root/dyn_private/data.sqlite:/usr/local/bin/data.sqlite -v /root/dyn_private/settings:/usr/local/bin/settings -v /root/dyn_private/maps:/usr/local/bin/maps -v /root/dyn_private/logs:/var/log --name dyn -d n0uk/dynast.io-server-private:release
```

#### Описание команды: 
* `--env-file /root/dyn_private/settings.env` — путь к вашему .env файлу
* `-e DYNASTIO_LABEL="Noahs-test"` — название вашего сервера
* `-p 80:80` — открытие портов сервера (поменяйте если используете другие)
* `-v /root/dyn_private/data.sqlite:/usr/local/bin/` — путь к дб, картам, настройкам и логам, поменяйте первую часть настройки на путь к вашим файлам вплоть до `/root/dyn_private/` (к примеру, если файлы привата находятся в папке dyn_private на диске C, путь будет C:/dyn_private)
* `--name dyn` — название docker контейнера

---

### Управление контейнером

* **Команда остановки контейнера:** 
  ```bash
  docker stop dyn
  ```
* **Команда перезагрузки контейнера:**
  ```bash
  docker restart dyn
  ```
  *(Где `dyn` - название вашего контейнера)*

* **Проверка запущенных контейнеров:**
  ```bash
  docker ps
  ```
